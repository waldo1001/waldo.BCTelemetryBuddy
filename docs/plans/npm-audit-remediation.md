## Task
Resolve all 23 vulnerabilities currently reported by `npm audit` at the workspace root (1 low, 7 moderate, 13 high, 2 critical) without changing public APIs.

## Scope boundary
- IN:
  - Bumping the two direct dependencies that ship vulnerable versions: `axios` (in `packages/shared` and `packages/extension`) and `express` (in `packages/mcp`).
  - Refreshing transitive deps via `npm audit fix` (no `--force`) at the workspace root: `undici`, `protobufjs`, `handlebars`, `hono`, `@hono/node-server`, `jws`, `lodash`, `underscore`, `markdown-it`, `qs`, `body-parser`, `path-to-regexp`, `picomatch`, `minimatch`, `brace-expansion`, `@isaacs/brace-expansion`, `ip-address`, `follow-redirects`, `ajv`, `express-rate-limit`, `@azure/functions`.
  - Re-running `npm run build` and `npm run test` for all three packages to confirm the bumps don't regress behavior.
  - Re-running the security-scan skill (Phase 8) so we leave with a clean audit.
- OUT:
  - Any major-version upgrades that `npm audit fix` would only do under `--force` (e.g. moving off `applicationinsights` or `@modelcontextprotocol/sdk`). If `audit fix` produces a clean report without `--force`, we stop there.
  - Code changes to consumers of `axios`/`express` beyond what a minor bump requires. If a bump is type-incompatible we revert and split into a separate plan.
  - Changes to lockfile policy, registry, or CI auditing thresholds.
  - New telemetry ‚Äî this is a dependency-hygiene chore, not a feature.

## Files to create / touch
- `package-lock.json` (root) ‚Äî regenerated by `npm audit fix`.
- `packages/shared/package.json` ‚Äî bump `axios` from `^1.6.0` to `^1.15.1` (latest patched line, see Risks).

## Risks
- `axios@1.15.1` patches a SSRF/NO_PROXY chain; if `BCTB_HTTPS_PROXY`/`NO_PROXY` env handling in `packages/shared` relied on the buggy normalization, behavior shifts. Low likelihood ‚Äî we set the proxy explicitly in the kusto client ‚Äî but call out for VERIFY.
- `undici` is bumped via `applicationinsights`'s nested `@azure/functions`. If `audit fix` deduplicates and the top-level `undici` moves to a major we don't support on Node 18, telemetry uplink could regress. Engines say Node ‚â• 18.18 in this repo; current undici 7.x supports that.
- `protobufjs` patch is critical (arbitrary code execution); the bump is forced by the OTel exporter chain. Risk of OTel exporter incompatibility is low because the patch is on the minor line.
- Lockfile churn is large (~70 packages touched). Reviewers should diff `package-lock.json` summary, not line-by-line.

## Blast radius / breakage prediction
- **Rating:** `low-risk`
  - No public API, tool schema, command, config key, event ID, or on-disk format changes. Pure dependency upgrades within the same major lines.
  - All advisories mark fixes as non-major (no `isSemVerMajor`). The dry-run completes without `--force`.
  - Behavior the user can observe (HTTP requests via axios, MCP HTTP server via express, telemetry via undici/protobufjs) keeps the same shape; only patched bug behavior changes.
- **Who/what could break:** MCP HTTP server consumers (express patch), kusto/auth callers (axios patch), telemetry pipeline (undici/protobufjs/handlebars in the OTel chain). Saved queries, KB cache, and event IDs are untouched.
- **Detection:** existing Jest suites in `packages/shared` (axios + auth), `packages/mcp` (express + tool handlers), and `packages/extension` (webview + axios) will fail loudly on a behavior regression. Post-merge, an MCP smoke (`/mcp` POST) and one real telemetry round-trip (`get_event_catalog`) confirm the hot paths. A regression in undici would surface as 5xx or hang on the MCP HTTP transport.

No migration path or version bump required ‚Äî this is a hygiene patch, not a behavior change.
